# opsdrill-decks

Public deck content for [opsdrill](https://opsdrill.dev). Every public flashcard on the site lives in this repo as YAML. Anyone can open a PR to add, fix, or improve a deck.

## What is this?

A deck is a collection of flashcards tied to a single Topic (e.g. Terraform, Kubernetes, Linux File Permissions). Decks are authored here, validated by CI, reviewed by the maintainer, and then imported into the opsdrill app. Users study them on the site or export to Anki (`.apkg`).

## Repo layout

```
topics/
  <topic-slug>/
    <deck-slug>.yml
```

The path encodes structure. There is no `topic_slug` or `slug` field inside the YAML — those come from the parent directory and filename. This rule prevents a whole class of drift bugs. See `CONTRIBUTING.md`.

## Files in this repo

| Path | Purpose |
|---|---|
| `topics/` | All deck YAML files, grouped by topic slug |
| `topics.json` | Machine-readable list of valid topic slugs. Generated by the opsdrill app. Do not hand-edit (the next app push will overwrite you). |
| `schemas/` | JSON Schema definition for deck files (`deck.v1.json`) |
| `scripts/` | Validators run by CI and locally |
| `.github/workflows/validate.yml` | CI: runs both validators on every PR |
| `CONTRIBUTING.md` | How to add or edit a deck |

## Contributing

See [CONTRIBUTING.md](./CONTRIBUTING.md).

Quick version:

1. Fork the repo.
2. Copy `topics/_template/example.yml` to `topics/<topic-slug>/<deck-slug>.yml`.
3. Edit the YAML.
4. Run validators locally: `pip install -r requirements.txt && python scripts/validate_layout.py && python scripts/validate_yaml.py`.
5. Open a PR.
